# Wall-X V1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a v1 Wall-X runtime on top of the existing async inference client/server flow so the client bootstraps a high-level plan from the first `fixed` frame, executes one 15-second subtask at a time, overlays only the `fixed` image with the training-style red dot, filters stale action chunks by `subtask_id`, and stops cleanly when all subtasks are complete.

**Architecture:** A new `WallXTaskOrchestrator` owns planner bootstrap, temporary image creation, ordered subtask generation, monotonic timeout tracking, fixed-camera overlay, and observation metadata injection. `robot_client.py` stays responsible for capture, queue management, and local action execution, while `policy_server.py` keeps the existing preprocessing/inference/postprocessing pipeline and only strips `__wallx_meta` before preprocessing plus wraps action chunks in a payload dict.

**Tech Stack:** Python 3.10, LeRobot async inference modules, gRPC transport, OpenCV, `wall_point.high_level`, pytest.
